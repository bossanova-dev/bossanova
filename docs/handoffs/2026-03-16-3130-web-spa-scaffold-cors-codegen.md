### Learnings & Notes

- Connect-ES v2 unified codegen: only `protoc-gen-es` is needed, `protoc-gen-connect-es` was removed. Service descriptors are generated by protoc-gen-es and consumed directly by @connectrpc/connect.
- buf.gen.yaml needs full path to protoc-gen-es binary: `services/web/node_modules/.bin/protoc-gen-es` since it's installed in the web package, not globally.
- TypeScript 5.9 defaults to `erasableSyntaxOnly: true` in Vite templates, but generated proto code uses TypeScript `enum` (non-erasable). Must set to `false`.
- CORS handler chain order: CORS wraps mux, auth wraps CORS — this ensures OPTIONS preflight passes through before hitting JWT validation.
- ConnectRPC web client requires `Connect-Protocol-Version` header in CORS AllowedHeaders.
- Generated TS files go to `services/web/src/gen/` (gitignored) — must run `buf generate` after clone.

### Issues Encountered

- protoc-gen-es not found in PATH when running `buf generate` from root — fixed by using full relative path in buf.gen.yaml
- TypeScript build failed with erasableSyntaxOnly error on generated enum types — fixed by disabling the option in tsconfig.app.json
