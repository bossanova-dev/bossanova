### Implementation Notes

- Plugin uses a manually-built `grpc.ServiceDesc` for TaskSourceService (matching daemon's HostService pattern) because the project uses connect-go, not protoc-gen-go-grpc.
- The GRPCPlugin bridge uses `broker.Dial(1)` to connect back to the daemon's HostService — this is the go-plugin bidirectional pattern where the daemon starts a broker server at ID 1 and the plugin dials it.
- `go mod tidy` does NOT work for this project (bossalib is local-only, not published to a remote). Use `go work sync` from the workspace root instead, which populates indirect deps correctly.
- Check aggregation logic mirrors daemon's poller: any failure → failed, all completed non-failure → passed, otherwise → pending.
- PollTasks classification: checks passed + mergeable → AUTO_MERGE; checks failed → CREATE_SESSION with plan describing the fix workflow; no checks / pending / not mergeable → skip (nil task, re-evaluated on next poll).
- The `isPreviouslyRejected` function is implemented but not yet wired into PollTasks — it needs a closed PR data source that the current HostService doesn't provide. Can be wired in when HostService adds ListClosedPRs or the daemon passes closed PR history.
- Tests use a `classifyPRWithMock` helper that duplicates the classification logic to test against a mock interface, since server.classifyPR directly calls the concrete hostServiceClient. A future refactor could extract an interface for cleaner testing.
