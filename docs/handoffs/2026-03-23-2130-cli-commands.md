### Learnings & Notes

- The BossClient interface is the abstraction layer — both LocalClient and RemoteClient implement it. Autopilot is local-only for now (RemoteClient returns errLocalOnly)
- Table formatting follows the existing `boss ls` pattern: bubbles/v2/table with views.CLITableStyles(), views.CLIColumnsWidth(), views.MaxColWidth()
- The `autopilot` command has an `ap` alias for convenience
- `list` subcommand also has `ls` alias matching the repo subcommand pattern
- Optional workflow ID resolution: when no ID is provided to status/pause/resume/cancel, resolveActiveWorkflow finds the single active workflow or errors with a helpful list if multiple
- StreamAutopilotOutput follows the same event oneof pattern as AttachSession — OutputLine for text, StatusUpdate for workflow state changes
- The existing stubClient in tests (newsession_test.go) is shared via composition — repoAddStubClient embeds it, so adding methods to stubClient fixes both test files
- `go build ./cmd/...` fails because `cmd` is both a directory and the build output — use `go build -o /tmp/binary ./cmd/` instead
