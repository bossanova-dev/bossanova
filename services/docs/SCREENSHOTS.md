# Screenshot inventory

Single source of truth for screenshots used by this docs site. Each
file is committed under its **final filename** so a real PNG can drop
in over the placeholder without touching markdown, sidebars, or CSS.

The current placeholders are deterministic, labeled PNGs generated by
`scripts/make-placeholder.py` (Python stdlib only; no PIL or sharp
required). They render distinct title + subtitle text per filename so
you can see at a glance which slot still needs a real screenshot.

## How to replace a placeholder with a real screenshot

1. Run `boss` in a sufficiently large terminal (~140×40).
2. Reach the state described in the "Should depict" column above.
3. Capture the terminal as a 1200×800 (or larger; 16:10 ratio) PNG.
4. Save **over** the placeholder at the same path under
    `static/img/screenshots/`; the filename is the final filename.
5. Move the row in this file from **Placeholders** to **Done**.
6. Commit.

## Regenerating a placeholder

If you ever want to regenerate the placeholders (e.g. after adding a
new screenshot slot):

```bash
cd services/docs
python3 scripts/make-placeholder.py \
  static/img/screenshots/<file>.png \
  "TITLE" "SUBTITLE DESCRIPTION"
```

Title is uppercase by convention. Background tint is derived from the
filename, so two different filenames yield two visually distinct
placeholders without any extra config.
